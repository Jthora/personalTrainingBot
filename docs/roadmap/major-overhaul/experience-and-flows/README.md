# Experience and Flows

## Objectives
- Map critical flows with mission framing and offline-aware behavior.

## Key Flows
- Home: readiness score, top two next actions, quick sync/offline indicator, low-data toggle.
- Mission Kit browse/select: filter/sort, deep-link behavior, offline cached fallback.
- Drill run: start, step navigation, success/abort handling, offline continuity, telemetry hooks.
- Signals: surface updates/coordination items, acknowledgements.
- AAR: capture outcome, lessons, export/share.

## Accessibility and Motion
- Motion minimal by default; prefer subtle progress states; provide reduced-motion compliance.

## Acceptance
- Each flow documented with entry/exit, states (loading/empty/offline/error), and UI copy.
- Offline states defined for Home, Kit detail, and Drill run.
