# Ops and Team

## Objectives
- Provide lightweight coordination (Signals) and after-action review.

## Signals Surface
- Items: assignments, status pings, alerts; ack/resolve states; timestamps.
- Views: list and drill-down; filters by role/tag.
- Offline: queue updates locally; sync later.

## Roles and Permissions (lightweight)
- Role tags on users/entries; no hard ACL in first drop—document assumptions and risks.

## AAR Workflow
- Template: context, actions taken, outcomes, lessons, follow-ups.
- Export/share: text/JSON; avoid sensitive data by default.
- Storage: local-first with optional sync.

## Acceptance
- Signals visible; items can be marked ack/resolve.
- AAR can be created, saved locally, and exported.
